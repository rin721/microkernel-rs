# 项目规则与规范 (AI 智能体工作流)

这里是本项目开发的**唯一真相来源 (Single Source of Truth)**。所有的规则、架构设计、开发流程都在此确立，任何开发工作（包括 AI 智能体和人类开发者）必须首先阅读并遵循本指南。

## 1. 核心规则与工作边界

*   **文档先行**：编写任何代码之前，必须查阅并更新 `docs/` 目录中的契约文档。代码必须 100% 遵守文档契约。
*   **层次化目录与防膨胀约束**：目录封装不要同级化设计。如果单文件代码量过多，就不适合阅读，应将目录结构拆分层次设计，严禁单文件代码积压膨胀。
*   **通用库代理约束与解耦**：通用库应用封装时须并保持通用性耦合设计，可随时替换底层库并与系统本体强制约束的生命周期进行约定绑定，并将其能力接口抽象绑定。在新增插件时，插件本体会自动映射出通用库应用的能力接口。
*   **长期自进化**：AI Agent 在编写代码的过程中，如果发现需要建立新的开发规则、补充新的设计，或者修正现有设计，必须**同步更新本规则及对应的设计文档**。

## 2. 推荐阅读顺序

为快速掌握本项目，请遵循以下阅读顺序：
1.  本文件 (`docs/ai/.rules.md`)：理解基本规范和规则。
2.  **[架构概览图](../architecture/graph.md)**：图形化总览整个微内核架构、生命周期和组件依赖（**非常重要：辅助 AI 理解系统的 Graph 就在这里**）。
3.  [核心需求与边界](../requirements/core.md)：系统宏观要实现的目标与特性。
4.  [微内核核心架构设计](../architecture/microkernel.md)：了解插件与应用的组装及生命周期机制。
5.  [代码目录架构设计](../architecture/directory_structure.md)：指导 Rust 代码的模块拆分与层级管理。
6.  深入示例：
    *   [数据库通用应用示例](../examples/database_app.md)
    *   [用户管理+权限管理插件示例](../examples/user_auth_plugin.md)
