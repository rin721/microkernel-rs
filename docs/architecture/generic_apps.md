# 基础设施通用应用设计 (Generic Infrastructure Apps)

本文档定义了开源 Admin 后端项目中核心的“基础设施通用应用”清单。

所有在此列出的通用库封装，都必须实现系统本体的 `AppLifecycle` 契约，并通过微内核的依赖自动注入映射给上层业务插件使用。这种松耦合的设计确保了即使未来替换底层 Crate，只要保持抽象能力接口不变，业务插件也完全不受影响。

## 1. 核心应用清单与技术栈选型

### 1.1 Web 服务端 (http)
*   **主流底层库**：`axum` + `tower` + `tower-http`
*   **抽象能力**：提供统一的高性能异步路由注册接口、CORS 配置支持、全局统一的错误处理中间件、静态文件托管服务。
*   **注入给插件的能力**：允许插件动态挂载自己的路由表 (Router) 到系统的 Web Server 容器中。

### 1.2 认证与会话 (auth)
*   **主流底层库**：`jsonwebtoken` (或安全性更高的 `paseto`)
*   **抽象能力**：Token 的生成 (Sign) 与解析验证 (Verify)。
*   **注入给插件的能力**：签发凭证、校验过期时间与非法伪造、解析载荷（Payload）。

### 1.3 权限访问控制 (rbac)
*   **主流底层库**：`casbin` (Rust 版)
*   **抽象能力**：灵活的访问控制策略计算模型，支持 RBAC 与 ABAC 扩展。
*   **注入给插件的能力**：实时鉴定主体 (Subject) 对资源 (Object) 的操作 (Action) 是否合法。

### 1.4 命令行解析 (cli)
*   **主流底层库**：`clap` (Derive 风格)
*   **抽象能力**：解析应用启动参数、子命令管理。
*   **注入给插件的能力**：获取启动时的全局命令行 flag（例如环境配置、自定义脚本触发指令）。

### 1.5 国际化 (i18n)
*   **主流底层库**：`rust-i18n` 或 `fluent-rs`
*   **抽象能力**：加载与解析多语言字典文件（YAML/JSON）。
*   **注入给插件的能力**：提供带变量插值的语言包翻译宏或接口。

### 1.6 日志与监控 (logger)
*   **主流底层库**：`tracing` + `tracing-subscriber` + `tracing-appender`
*   **抽象能力**：多目标的异步非阻塞日志记录、结构化上下文追踪 (Span)。
*   **注入给插件的能力**：按日滚动、按大小分割的文件归档能力，以及全链路的跨组件 TraceID 穿透。

### 1.7 关系型数据库 (database)
*   **主流底层库**：`sea-orm` (基于 `sqlx` 构建的纯异步动态 ORM)
*   **抽象能力**：统一的数据库连接池生命周期维护、数据库迁移 (Migration) 管理。对于 Admin 系统，`sea-orm` 能极大简化多表关联查询、动态条件过滤与分页统计。
*   **注入给插件的能力**：透明可用的异步 SQL 会话 (DatabaseConnection)，支持强类型的实体 CRUD 操作。

### 1.8 缓存与会话状态 (cache)
*   **主流底层库**：`redis` (分布式异步缓存) + `moka` (高性能本地进程内缓存)
*   **抽象能力**：支持本地与分布式两级缓存架构，统一管理高可用 Redis 连接。
*   **注入给插件的能力**：提供统一的 KV 存取接口（支持 TTL 过期），用于缓存热点数据、RBAC 权限树、用户登录状态 (Session)。

### 1.9 统一存储 (storage)
*   **主流底层库**：`opendal` (Apache OpenDAL)
*   **抽象能力**：将本地文件系统、S3、阿里云 OSS、腾讯云 COS 等各类云存储的 API 抽象为同一套读写语义。
*   **注入给插件的能力**：无缝的跨存储后端的文件上传、下载、删除操作接口，业务逻辑与具体云厂商解耦。

### 1.10 邮件服务 (mail)
*   **主流底层库**：`lettre`
*   **抽象能力**：SMTP 连接管理与异步发送支持。
*   **注入给插件的能力**：组装邮件模板内容，触发并监控异步邮件发送状态。

### 1.11 密码学安全 (password/crypto)
*   **主流底层库**：`argon2` + `rand` (RustCrypto 生态)
*   **抽象能力**：抵抗侧信道攻击的最高安全级别密码 Hash 算法抽象。
*   **注入给插件的能力**：对敏感字段加密与哈希比对能力。

### 1.12 多因素认证 (mfa)
*   **主流底层库**：`totp-rs` + `qrcode`
*   **抽象能力**：Time-based One-Time Password 核心算法。
*   **注入给插件的能力**：动态生成密钥串，提供供验证器（如 Google Authenticator）扫码绑定的二维码数据流，并在登录阶段校验 6 位动态码。
