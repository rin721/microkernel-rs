# 微内核核心架构设计 (Microkernel Architecture)

本项目采用 Rust 后端微内核架构设计，其核心思想是“应用依赖组装注入”和“插件化”。

## 1. 架构目标

核心目标是将系统划分为“内核 (Kernel)”和“插件/应用 (Plugins/Apps)”两部分：
*   **微内核**：负责依赖组装、生命周期管理、配置分发、统一启停以及资源释放。
*   **插件与应用**：负责具体的业务逻辑，通过内核定义的接口规范进行注册和相互通信。

## 2. 核心设计：应用依赖组装注入

每个应用（如数据库应用、HTTP服务等）都需要完整的生命周期与依赖管理。
组装过程包含：

*   **配置重载 (Config Reload)**：支持动态更新或启动时加载应用配置。
*   **初始化 (Initialization)**：实例化所需资源。
*   **中间件 (Middleware)**：在应用外层挂载横切关注点（如日志、追踪、鉴权）。
*   **统一启动与停止**：内核统一调度所有注册应用的启动，并在终止时安全、有序地进行资源关闭。

## 3. 插件系统设计 (Plugin System)

系统中的模块交互以插件的形式组织：

*   **生命周期管理**：
    *   `Load`: 插件加载，进行资源预分配和依赖检查。
    *   `Start`: 插件启动，开启后台任务和监听端口。
    *   `Destroy`: 插件销毁，执行资源清理回收。
*   **通信与共享协议**：
    *   **接口/Trait**：插件间通过明确定义的 Rust Trait（动态分发 `Arc<dyn Trait>` 优先，热路径考虑静态）进行交互。
    *   **事件总线 (Event Bus)**：用于解耦不同插件间的异步消息传递。

## 4. 依赖注入与挂载

系统本体（Host）将充当代理，调用插件/应用的特定钩子（Hooks）来完成挂载。参见 [数据库通用应用示例](../examples/database_app.md) 获取更详细的钩子规范。
