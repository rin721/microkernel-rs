# 数据库通用应用设计示例 (Database App Example)

本示例展示如何将一个数据库库（例如基于 SQLx 的 PostgreSQL 客户端）封装为微内核架构下的通用应用，并实现完整的生命周期管理。

## 1. 目标与定位

*   **职责**：封装数据库连接池，管理与数据库的交互。
*   **接入方式**：作为“通用应用库”，向系统本体注册其生命周期钩子，从而由微内核统一管理。

## 2. 生命周期钩子实现规范

任何通用库如果希望接入微内核，必须实现系统本体定义的一系列 Trait/Hooks：

### 2.1 默认配置接口 (Default Config)
*   **功能**：为当前数据库应用提供一套合理的默认配置（如最大连接数、默认超时时间等）。
*   **工作机制**：只需实现该接口，系统本体在启动时会通过“启用字段”判断是否自动代理调用。如果用户配置中该应用已启用但缺失某些字段，系统将使用此默认配置进行覆盖。

### 2.2 实例创建前接口 (Pre-Create Hook)
*   **功能**：在数据库连接池对象实例化之前执行。
*   **应用场景**：用于校验配置项的合法性，或从密码中心拉取动态密码。系统本体会调用此钩子。

### 2.3 实例创建后接口 (Post-Create Hook)
*   **功能**：在数据库连接池对象实例化完成之后立即执行。
*   **应用场景**：用于打印日志“数据库连接池已初始化”，或进行一次基础的 ping 测试以确认网络通畅。

### 2.4 实例挂载前接口 (Pre-Mount Hook)
*   **功能**：在数据库实例即将被挂载到全局注册表/依赖注入容器前执行。
*   **应用场景**：用于执行数据库的自动迁移 (Migrations) 脚本，确保数据库 Schema 达到最新状态。

### 2.5 实例挂载后接口 (Post-Mount Hook)
*   **功能**：数据库实例已经完全准备好，并已在内核注册。
*   **应用场景**：开始接受来自其他业务插件的连接请求。

### 2.6 配置重载后接口 (Post-Reload Hook)
*   **功能**：系统配置管理数据发生变化时被调用。
*   **工作机制**：系统本体会传入 Diff 信息，应用判断变化的配置是否属于当前“数据库应用”。如果是，则触发连接池动态调整或连接字符串更新（甚至重新走一遍创建流程）。

### 2.7 实例停止前接口 (Pre-Stop Hook)
*   **功能**：系统接收到退出信号，即将关闭数据库实例前调用。
*   **应用场景**：向外发出拒绝新请求的信号，等待仍在进行中的长事务执行完毕（优雅停机）。

### 2.8 实例停止后接口 (Post-Stop Hook)
*   **功能**：数据库实例资源彻底释放后调用。
*   **应用场景**：记录“数据库连接已断开”的日志。
